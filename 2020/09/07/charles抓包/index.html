charles抓包

发表于 2020-09-07 | 分类于 charles

一、

安装前提：配置好了Jdk环境

若不进行破解，则每次只能试用30分钟，30分钟后工具自动关闭

破解方法：将破解补丁charles.jar文件复制到lib文件夹下，替换掉同名jar包

二、

工具简单介绍：

1：清空列表所有请求

2：开始抓取 / 停止抓取

3：开启节流 / 停止节流

4：开启断点 / 结束断点

5：编辑请求

6：重发请求

7：校验请求的结果

8：快捷工具箱

9：快捷设置

Structure：

以域名划分请求信息，可清晰地看到请求的数据结构

Sequence：

可清晰看到全部请求，不用一层一层点开，按数据请求的时间顺序执行

三、

快捷搜索功能：通过ctrl+F进入搜索页面

1、关键词输入框：可输入正则表达式，不必区分大小写

2、搜索路径：

（1）全部会话

（2）当前会话

（3）指定的请求路径，在右侧输入框中输入

3、搜索范围：

（1）请求URL

（2）请求Header

（3）请求Body

（4）返回Header

（5）返回Body

4、搜索结果显示

四、抓取手机请求

1、使手机与电脑处于同一局域网下（如连同一WiFi），如图，连接网络tczc，连接成功后打开代理设置，设置为【手动】

2、windows+R，输入cmd进入命令行，使用ipconfig命令查看电脑的ip地址

3、在服务器一栏中填入电脑ip地址，端口号填8888。（端口号在Proxy-Proxy Settings下自己设置的）

4、注意：首次设置代理网络时，如果电脑端的Charles是打开的，则会提示是否允许设备连接Charles，这时必须应许，否则无法抓包，手机也无法连接网络；若这样设置后还是无法上网，则尝试重启电脑+手机。

如果不小心点了Deny，可以到Proxy - Access Control Settings下去查看，可手动移除或新增允许访问的IP地址。

至此代理已经设置成功，可以抓取手机端的http请求。

5、若想抓取手机端的https请求，则必须安装手机端证书。

方法：在手机浏览器中输入 http://charlesproxy.com/getssl 去安装证书。

ios系统的，在设置 - 通用 - 描述文件与设备管理下可查看安装的证书。

另外要注意的是，在通用 - 关于本机 - 证书信任设置中，要开启对此证书的信任，否则同样抓取不到https请求。

6、以上设置完毕后，可以试着访问某个APP，如图，虽然能看到发出的请求，但请求和响应中出现了乱码：

解决方案是：

Proxy - SSL Proxying Settings下，将出现乱码的该域名添加进去

设置完毕后，清除所有会话，重新访问一次，当当当当！！！成功了！！

五、抓取电脑请求

1、抓取电脑http请求：勾选这一项即可 Proxy - Windows Proxy

2、抓取电脑端https请求，同样需要安装证书。

电脑端证书安装：Proxy - SSL Proxying - Install Charles Root Certificate

六、其他功能点（选中会话，点击右键）

1、Repeat：重复执行请求

2、Repeat Advanced：可以指定请求的遍数

3、Focus

右键某会话，点击Focus，可以使该域名置顶，其他域名统一被放到下面的Other Hosts下：

除了通过这种方法设置，也可在View - Focused Hosts下统一设置

4、黑名单 / 白名单

Black List：黑名单中的域名不能联网

White List：白名单开启后，只有在白名单中的域名可以访问网络，不在白名单中的不能联网

（如果一个域名既在黑名单，又在白名单，那么也不能联网）

5、导出会话功能

Export：可将会话导出到本地，下次可通过File - Open Session打开

6、比较功能

Compare：在左侧列表中选中两个请求，右键即可看到此选项，可用于比对两个请求的入参和出参
